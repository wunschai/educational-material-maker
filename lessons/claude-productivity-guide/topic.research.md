## 核心概念

### 1. 範式轉移：從對話問答到自主協作

AI 已從被動回答問題的聊天機器人，轉變為能理解任務目標、拆解步驟、調用工具並交付成果的數位代理人。對教師而言，這代表 AI 不再只是「幫你查資料、寫摘要」的工具，而是能協助備課全流程——從蒐集文獻、整理教學大綱、製作簡報到生成教學影片。對研究人員而言，AI 能承接文獻回顧、資料整理、圖表生成與報告撰寫等重複性高的工作，讓人專注在假設建構、實驗設計與批判性分析。

### 2. 四層架構：漸進式揭露的工作系統

Claude 採用分層設計：Chat（大腦）負責戰略規劃與邏輯辯證；Skill（知識）將專業 SOP 模組化為可重複調用的「數位食譜」；MCP（手腳）連接外部工具與資料庫，打破資訊孤島；Cowork/Code（執行者）在受控環境中交付實際成果。這種分層讓 AI 依任務需求逐層啟用，降低上下文負擔並便於權限管理。

### 3. Chat 模式：教師與研究者的思考夥伴

Chat 不只是問答介面，而是知識工作的「邏輯整形層」。教師可用它承接三類任務：課程設計（將教學目標轉為教案架構）、教材比較（不同教學策略的利弊分析）、知識整理（將零散文獻轉為系統化講義）。研究人員則可用來釐清研究問題、比較方法論、整理文獻回顧骨架。模型選擇上，Opus 4 適合深度學術推理與複雜教案設計，Sonnet 4 是日常備課與寫作的主力，Haiku 4 處理批量作業評閱、資料分類等高頻任務。

### 4. Cowork 模式：教師的數位工作助理

Cowork 讓 AI 從「給建議」進化到「幫你做」。教育場景中的典型應用：翻拍紙本學員名冊自動轉為 Excel、將手寫筆記整理成結構化文件、批量處理學生作業格式、定時生成每週教學簡報、遠端交辦格式轉換與檔案上傳。核心能力包括環境隔離（保護學生資料安全）、多模態處理（紙本→數位自動轉換）、子代理並行（同時搜尋多個教學資源網站）、以及自動化排程（每週固定產出教材）。

### 5. Code 模式與 Skill/MCP 的教育應用

Code 模式讓教師能用自然語言操控檔案系統與自動化流程，即使不熟悉程式語言也能建立教學工具（如本教材製作 plugin 就是用 Code 模式開發的）。Skill 將教學 SOP 模組化——例如把「從主題到教學影片」的五階段流程封裝為可重複調用的技能包。MCP 作為開源連接標準，讓 AI 接入學術資料庫（arXiv、Google Scholar）、教學平台（Moodle）、文件系統（Google Drive）與筆記工具（Notion），打通教學與研究工作中的資訊孤島。

### 6. 教育工作者的 AI 工作系統建構

對教師與研究人員而言，真正的效率提升不在於用了哪個模型，而在於是否建立了一套「可重複、可累積、可分享」的 AI 工作流。這需要：清楚區分思考層（Chat 釐清教學目標）/ 知識層（Skill 封裝備課流程）/ 工具層（MCP 接入教學資源）/ 執行層（Cowork/Code 產出教材）。當這套系統跑通，教師的角色從「親手做每一件事」轉變為「設計流程、審核品質、專注教學」——AI 承接重複性的數位勞務，人專注在只有人能做的教學判斷與師生互動。

## 常見誤解

- **「AI 就是聊天機器人」**：Claude 代理系統能主動規劃、跨工具協作與交付成果，已超越傳統問答範疇。
- **「用最強的模型就好」**：模型選擇應依任務特性分層（Opus 深度推理 / Sonnet 日常主力 / Haiku 高頻篩選），而非一律選最強。
- **「Skill 就是更長的 prompt」**：Skill 採三層漸進式結構控制 token 效率，是可治理、可共用的能力模組，非單次提示詞。
- **「有了 MCP 就不需要 Skill」**：兩者互補——MCP 解決「做不做得到」（工具連接），Skill 解決「會不會做」（方法封裝）。
